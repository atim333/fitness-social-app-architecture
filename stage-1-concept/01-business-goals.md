# 1. Бизнес-цели проекта

## Контекст
Компания производит спортивные товары и стремится освоить новые каналы информирования покупателей, стимулировать спрос на новую продукцию через создание социального фитнес-приложения.

## Ключевые бизнес-цели (4 цели, согласно требованиям)

### 1. Увеличение вовлеченности пользователей и времени взаимодействия с брендом
- **Цель:** Увеличить ежемесячное активное использование приложения до 1.5 млн пользователей в течение 2 лет
- **Метрики:** 
  - MAU (Monthly Active Users) > 1.5M через 24 месяца
  - Среднее время в приложении > 15 минут в день
  - Retention rate (удержание) > 40% на 90-й день
- **Влияние на бизнес:** Прямая связь с узнаваемостью бренда и лояльностью
- **Технические импликации (из курсовой):**
  - Многоуровневое кэширование для мгновенной загрузки соц. ленты
  - Eventual consistency для социальных функций (приемлема кратковременная рассинхронизация)
  - Redis для хранения активных сессий пользователей

### 2. Стимулирование продаж через интегрированные промоакции
- **Цель:** Увеличить конверсию просмотра товаров в покупки на 25% через интеграцию с приложением
- **Метрики:**
  - CTR (Click-through rate) промоакций > 8%
  - Конверсия в продажи из приложения > 3.5%
  - Рост продаж новых коллекций через приложение на 30%
- **Влияние на бизнес:** Прямой ROI от разработки приложения
- **Технические импликации (из курсовой):**
  - Строгая согласованность (ACID) для промоакций и бронирования
  - Event-driven обновление цен на всех витринах
  - Сессии фиксации промо-цен на ограниченное время

### 3. Создание устойчивого комьюнити и социального влияния
- **Цель:** Сформировать 50,000 активных социальных групп по интересам в течение 18 месяцев
- **Метрики:**
  - Количество созданных групп > 50,000
  - Средний размер группы > 15 участников
  - Активность в группах (сообщения/день) > 5
- **Влияние на бизнес:** Органический рост, снижение затрат на маркетинг
- **Технические импликации (из курсовой):**
  - PostgreSQL для социальных графов (реляционная целостность)
  - Redis для кэширования ленты новостей и активных чатов
  - Двухфазное резервирование для групповых событий

### 4. Укрепление позиции бренда как технологического лидера
- **Цель:** Достичь рейтинга приложения > 4.5 звезд в основных app stores
- **Метрики:**
  - App Store/Google Play rating > 4.5
  - Положительных отзывов > 85%
  - Упоминаний в спортивных медиа > 100 в год
- **Влияние на бизнес:** Усиление конкурентного преимущества
- **Технические импликации (из курсовой):**
  - Высокая доступность (99.9% uptime) через мультиоблачную архитектуру
  - Защита данных (GDPR/HIPAA compliance)
  - Минимальные задержки через географически распределенные кэши

## Связь бизнес-целей с требованиями

| Бизнес-цель | Соответствующие требования из ТЗ | Технические решения из курсовой |
|-------------|-----------------------------------|----------------------------------|
| 1. Вовлеченность | 1, 2, 5, 7, 8 | Многоуровневое кэширование, eventual consistency, Redis сессии |
| 2. Продажи | 10, 11 | ACID транзакции, event-driven обновление, ценовые сессии |
| 3. Комьюнити | 1, 3, 6 | PostgreSQL соц. графы, Redis кэши, двухфазное резервирование |
| 4. Имидж бренда | Все, особенно 4, 9 | Высокая доступность, безопасность, геораспределение |

## Архитектурные требования, вытекающие из бизнес-целей

### Из цели 1 (Вовлеченность):
- **Производительность:** < 2 сек загрузка основных экранов
- **Согласованность:** Eventual consistency приемлема для соц. функций
- **Масштабируемость:** Поддержка 1.5M+ активных пользователей

### Из цели 2 (Продажи):
- **Надежность:** Гарантированное проведение транзакций
- **Согласованность:** Строгая согласованность для финансовых операций
- **Интеграция:** Бесшовная связь с e-commerce системами

### Из цели 3 (Комьюнити):
- **Доступность:** 24/7 доступ к социальным функциям
- **Производительность:** Реальная-time обновление ленты
- **Целостность данных:** Предотвращение потери сообщений

### Из цели 4 (Имидж бренда):
- **Безопасность:** Соответствие GDPR, HIPAA, локальным требованиям
- **Наблюдаемость:** Полный мониторинг и логирование
- **Отказоустойчивость:** Минимальное время восстановления

## Ожидаемые бизнес-результаты через 3 года
1. Рост общей выручки компании на 8-12% за счет приложения
2. Снижение стоимости привлечения клиента на 15%
3. Увеличение lifetime value клиента на 20%
4. Позиционирование компании как инновационной в digital-сфере

## Ключевые архитектурные решения (предварительно, на основе курсовой)
1. **Дифференцированный подход к согласованности:**
   - Строгая согласованность для: платежей, бронирования, промоакций
   - Eventual consistency для: соц. ленты, рекомендаций, лидербордов

2. **Стратегия управления данными:**
   - PostgreSQL для ACID-транзакций (Identity, Payments, Events)
   - MongoDB для документ-ориентированных данных (Workouts, Social Posts)
   - Redis для кэширования и временных данных (Sessions, Leaderboards)

3. **Паттерны обработки транзакций:**
   - Двухфазное резервирование для бронирования мест
   - Сага-паттерн для распределенных бизнес-процессов
   - Компенсирующие транзакции для отката операций
